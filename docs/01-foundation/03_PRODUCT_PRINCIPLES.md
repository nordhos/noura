# NOURA Product Principles

Version: 1.0

Status: LOCKED

---

# Purpose

Product Principles exist to guide every product decision.

Whenever multiple solutions are possible, these principles determine which solution best represents NOURA.

If a proposed feature conflicts with these principles, the principles take precedence.

---

# Principle 01

## Financial Clarity First

The purpose of every feature is to increase users' understanding of their financial condition.

If a feature adds functionality without improving clarity, it should be reconsidered.

Ask:

"Will users better understand their financial condition after using this feature?"

---

# Principle 02

## Trust Is More Important Than Convenience

Financial information must always be accurate.

Never sacrifice correctness for convenience.

Users should always trust the numbers shown in NOURA.

---

# Principle 03

## Simplicity Wins

When two solutions solve the same problem:

Choose the simpler one.

Simple products are easier to understand.

Simple products are easier to trust.

Simple products are easier to maintain.

---

# Principle 04

## Every Screen Answers One Question

Every screen has one primary purpose.

Dashboard

"What is my financial condition?"

Transactions

"What happened?"

Reports

"How has my financial journey evolved?"

Settings

"How is my financial system configured?"

Avoid mixing multiple purposes into one screen.

---

# Principle 05

## Every Input Must Create Immediate Value

Users should never feel they are filling forms.

Every input should immediately improve Financial Clarity.

Examples

Financial Setup

↓

Creates the financial system.

Initial Balance

↓

Creates the first financial transaction.

Transaction

↓

Updates Dashboard, Reports and Financial Health.

---

# Principle 06

## Explain Before Display

Numbers alone rarely create understanding.

Whenever possible:

Explain

before

Visualize.

Context creates confidence.

---

# Principle 07

## Opinionated Products Build Better Experiences

NOURA intentionally supports only the workflows it believes create the best experience.

Not every configuration should be configurable.

Not every edge case should become a feature.

Focus creates quality.

---

# Principle 08

## Build For Daily Confidence

People open NOURA because they want certainty.

Every interaction should leave users feeling more confident than before.

Never create anxiety through unnecessary complexity.

---

# Principle 09

## Product Before Technology

Technology serves the product.

Architecture serves the product.

Code serves the architecture.

Never reverse this order.

---

# Principle 10

## Long-Term Consistency

Every new feature should feel like it has always belonged in NOURA.

Users should experience one coherent system, not a collection of unrelated features.

Consistency is one of NOURA's greatest competitive advantages.

---

# Decision Checklist

Before approving any feature, answer these questions.

□ Does this improve Financial Clarity?

□ Does this strengthen user trust?

□ Does this simplify the experience?

□ Does this reduce uncertainty?

□ Does this align with NOURA's product vision?

If the answer to any question is "No", revisit the solution before implementation.

---

# Product Standard

NOURA does not measure success by the number of features.

NOURA measures success by one outcome:

People understand their financial condition better after using the product.

Every product decision should move users closer to Financial Clarity.
