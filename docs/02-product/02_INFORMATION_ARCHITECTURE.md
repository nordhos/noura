# NOURA Information Architecture

Version: 1.0

Status: LOCKED

---

# Purpose

Information Architecture defines how financial information is organized inside NOURA.

It focuses on information, not screens.

Every screen is built upon this structure.

---

# Core Information Model

Financial System
│
├── Dashboard
├── Transactions
├── Reports
├── Insights
└── Settings

---

# Dashboard

Purpose

Understand today's financial condition.

Contains

- Current Balance
- Total Income
- Total Expense
- Recent Transactions
- Quick Actions

Primary Question

"How am I doing today?"

---

# Transactions

Purpose

Capture financial activities.

Contains

- Income
- Expense
- Categories
- History

Primary Question

"What happened?"

---

# Reports

Purpose

Understand historical financial performance.

Contains

- Monthly Summary
- Category Analysis
- Income Analysis
- Expense Analysis

Primary Question

"What has happened over time?"

---

# Insights

Purpose

Explain financial behavior.

Contains

- Spending Patterns
- Financial Health
- Recommendations
- Trends

Primary Question

"What should I learn?"

---

# Settings

Purpose

Configure the financial system.

Contains

- Profiles
- Categories
- Financial Timeline
- Preferences

Primary Question

"How is my financial system configured?"

---

# Navigation Principle

Users should never wonder where information belongs.

Every piece of information should have one obvious place.

---

# Architecture Principle

Information comes before interface.

Structure comes before layout.

Meaning comes before visuals.
