# NOURA Design System

Version: 1.0

Status: Living Document

---

# Purpose

The Design System ensures every part of NOURA feels like one cohesive product.

It defines not only how NOURA looks, but also how it behaves.

Consistency builds trust.

Trust creates Financial Clarity.

---

# Experience Principles

Every interaction inside NOURA should feel:

- Calm
- Clear
- Predictable
- Reliable
- Fast
- Minimal

Users should never feel overwhelmed.

Users should always know what to do next.

---

# Design Philosophy

Clarity over decoration.

Content over visuals.

Meaning over aesthetics.

Consistency over creativity.

Simple interfaces create confident users.

---

# Layout Principles

Mobile First

Single-hand friendly

One primary action per screen

Consistent spacing

Consistent navigation

Predictable scrolling

---

# Navigation

Bottom Navigation

Primary destination only.

Maximum:

5 destinations.

Every destination answers a different question.

---

# Typography

Typography exists to create hierarchy.

Never decoration.

Priority:

Heading

↓

Metric

↓

Body

↓

Caption

---

# Colors

Colors communicate meaning.

Not decoration.

Primary

Brand identity.

Success

Positive financial outcome.

Warning

Needs attention.

Danger

Requires immediate attention.

Neutral

Supporting information.

---

# Icons

Icons support text.

Icons never replace text.

Users should never guess an icon's meaning.

---

# Components

Core Components

- Button
- Card
- Metric Card
- Input
- Bottom Sheet
- FAB
- Navigation Bar
- Dialog
- Toast
- Empty State
- Loading State

---

# Motion

Motion should explain.

Never distract.

Animation should communicate:

- Navigation
- Feedback
- Success
- Progress

Duration should feel natural.

---

# Empty States

Every empty state should answer:

Why is this empty?

What should I do next?

Never leave users without direction.

---

# Error States

Every error should:

Explain what happened.

Explain why.

Explain how to fix it.

Avoid technical language.

---

# Loading States

Loading should reassure users.

Use skeleton loading whenever possible.

Avoid unnecessary spinners.

---

# Accessibility

Readable typography.

Comfortable touch targets.

Sufficient color contrast.

Clear interaction feedback.

---

# Design Quality Checklist

Before approving a screen:

□ Is the purpose obvious?

□ Is the primary action obvious?

□ Can users understand it within five seconds?

□ Does this reduce cognitive load?

□ Does this improve Financial Clarity?
