# NOURA Copywriting Guidelines

Version: 1.0

Status: Living Document

---

# Purpose

Copywriting defines how NOURA communicates with users.

Every sentence should reduce uncertainty.

Every word should increase confidence.

The goal is not to sound clever.

The goal is to make users feel informed, guided, and in control.

---

# Brand Voice

NOURA is:

- Calm
- Clear
- Helpful
- Honest
- Encouraging

NOURA is never:

- Judgmental
- Dramatic
- Technical
- Robotic
- Overly casual

---

# Writing Principles

## Be Clear

Use simple language.

Prefer familiar words.

Avoid jargon.

---

## Be Direct

Tell users exactly what will happen.

Avoid vague wording.

Good

Save Transaction

Bad

Submit

---

## Be Helpful

Every message should help users take the next step.

Never stop at explaining a problem.

Always provide direction.

---

## Be Positive

Focus on what users can do.

Avoid negative wording whenever possible.

Instead of

"You entered an invalid amount."

Use

"Please enter a valid amount."

---

## Be Consistent

Use the same terms throughout the product.

Refer to the Ubiquitous Language document.

---

# Button Labels

Prefer action-oriented labels.

Examples

Save Transaction

Add Income

Add Expense

Create Profile

Continue

Finish Setup

View Report

Edit Profile

---

Avoid generic labels.

Avoid:

OK

Submit

Yes

Done

---

# Screen Titles

Every title should answer:

"What can I do here?"

Examples

Dashboard

Transactions

Reports

Settings

Add Expense

Add Income

Profile

---

# Empty States

Every empty state should contain:

1. What happened
2. Why it happened
3. What to do next

Example

No transactions yet.

Start by recording your first income or expense.

---

# Success Messages

Keep them short.

Examples

Transaction saved.

Profile updated.

Setup completed.

Changes saved.

---

# Error Messages

Every error should answer:

What happened?

Why?

What should the user do?

Example

Unable to save the transaction.

Please check your internet connection and try again.

Avoid

Unknown Error

500 Internal Server Error

Something went wrong.

---

# Confirmation Messages

Use confirmations only for actions that:

- Cannot be undone
- May cause data loss
- Have significant impact

Avoid unnecessary confirmations.

---

# Numbers

Always use localized formatting.

Currency

Rp 125.000

Dates

23 July 2026

Percentages

32%

---

# Microcopy Checklist

Before approving any text:

□ Is it easy to understand?

□ Is it action-oriented?

□ Does it reduce uncertainty?

□ Is it consistent with other screens?

□ Does it support Financial Clarity?
